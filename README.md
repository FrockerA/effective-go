# Subscriptions REST API

Микросервис для агрегации и управления данными об онлайн-подписках пользователей. Разработан на Go с использованием чистой архитектуры и покрыт интерактивной Swagger-документацией.

##  Возможности 
- **Полный CRUDL** для управления подписками (Создание, Чтение, Обновление, Удаление, Список с фильтрацией).
- **Продвинутая бизнес-логика**: Расчет суммарной стоимости активных подписок пользователя за выбранный период (с учетом пересечения дат).
- **Автоматические миграции** при старте приложения (`golang-migrate`).
- **Структурированное логирование** всех HTTP-запросов и ошибок с помощью `uber-go/zap`.
- **Контейнеризация**: Многоэтапная сборка (Multi-stage build) Docker для минимизации размера образа и `docker-compose` для поднятия инфраструктуры одной кнопкой.

##  Стек технологий
- **Язык**: Go (Golang)
- **База данных**: PostgreSQL 16
- **Роутер**: `go-chi/chi`
- **Работа с БД**: `jmoiron/sqlx`
- **Логирование**: `uber-go/zap`
- **Документация**: `swaggo/swag`
- **Инфраструктура**: Docker, Docker Compose

##  Быстрый старт 

### Требования
Для запуска проекта на вашей машине должен быть установлен [Docker](https://www.docker.com/) и Docker Compose.

### Шаг 1: Клонирование и конфигурация
Склонируйте репозиторий и создайте файл переменных окружения:
```bash
git clone <url_репозитория>
cd effective-go
cp .env.example .env
```
### Шаг 2: Запуск через Docker
Выполните сборку и запуск контейнеров (БД + API) одной командой:

``` Bash
docker compose up --build
```
При первом запуске автоматически применятся SQL-миграции и создастся таблица subscriptions.

##  Документация API (Swagger)
После успешного запуска сервера, интерактивная документация доступна в браузере по адресу:
 http://localhost:8080/swagger/index.html

Через Swagger UI вы можете ознакомиться со всеми эндпоинтами, структурами запросов и протестировать API в реальном времени.

Основные эндпоинты:
POST /subscriptions/ — Создать новую подписку.

GET /subscriptions/ — Получить список подписок (с поддержкой фильтров user_id и service_name).

GET /subscriptions/{id} — Получить подписку по ID.

PUT /subscriptions/{id} — Обновить данные подписки.

DELETE /subscriptions/{id} — Удалить подписку.

GET /subscriptions/total — [Бизнес-фича] Расчет стоимости подписок за период (формат дат MM-YYYY).
